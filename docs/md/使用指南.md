# 单资产 Merton-Kelly 量化策略控制台使用指南

本指南旨在指导用户如何利用 **Merton-Kelly 量化策略控制台** 执行一套严谨的“诊断-选品-计算-风控”投资流程。本工具将均值回归理论 (OU Process) 与凯利公式 (Kelly Criterion) 结合，为您提供基于数学概率的 LEAPS 期权仓位建议。

## 1. 启动与访问

您可以通过以下两种方式进入策略控制台：

* **Web 在线版**：直接访问 [https://kelly-zh.streamlit.app/](https://kelly-zh.streamlit.app/)
* **本地运行版**：
    确保环境已配置，在终端运行：
    ```bash
    streamlit run code/app_zh.py
    ```

---

## 2. 核心工作流 (S.O.P.)

本系统设计为一套标准化的作业流程，建议按顺序执行：

### Step 0: 市场诊断 (Diagnosis)
**核心目的**：验证均值回归假设是否成立，确认入场时机。

1.  **输入代码**：在侧边栏输入 Ticker（如 NVDA），系统会自动拉取数据。
2.  **核心验证**：
    * **PE 偏离度**：当前 PE 是否位于历史均线下方？（确认低估）
    * **Lambda (回归动力)**：观察历史走势图。如果当前 Lambda 处于历史极高位（例如 >90% 分位），说明近期波动造成了参数虚高。**此时务必在侧边栏手动调低 Lambda**，以防模型给出过于激进的预期。
3.  **分布推演**：查看下方的“未来价格路径分布锥”。它展示了基于当前参数，未来 3-6 个月触及“止损底线”或“盈利目标”的概率分布。

### Step 0.5: 最优期限求解 (The Solver)
**核心目的**：在众多期权到期日中，找到数学期望最高的那个。

1.  **设定补仓价 ($V_{fill}$)**：这是您的战术底线。假设股价跌到此价格，您计划打满子弹。
2.  **求解逻辑**：系统会寻找一张“完美的合约”，使得：
    * **当前 ($P$)**：应用起始 $k$ 值（如 0.5）时，仓位适中。
    * **触底 ($V_{fill}$)**：应用满仓 $k$ 值（如 1.0）时，建议仓位恰好达到 100%。
3.  **结果应用**：记录系统推荐的“到期日天数”，在券商软件中寻找最接近该日期的合约。

### Step 1: 主仓位计算器 (The Dashboard)
**核心目的**：基于选定合约，计算具体的买入金额和张数。

1.  **数据回填**：将在券商软件中查到的合约数据（最新 Ask Price, Delta, Theta）填入侧边栏的 **2.2 市场与合约参数**。
2.  **关键指标检查**：
    * **ERP (净优势)**：必须 > 0。如果 ERP 为负，说明期权的时间损耗和资金成本超过了理论回归收益，**禁止开仓**。
    * **Alpha (信心系数)**：反映股价在 $[V_{hard}, V_{target}]$ 区间的位置。数值越接近 1.0，表示越接近硬底，建议仓位越重。
3.  **动态推演**：
    * **仓位路径图**：展示了股价若继续下跌，应如何分批加仓（网格交易）。
    * **压力测试**：展开页面底部的风险表，确认“如果明天发生 3σ 级别的暴跌，账户亏损额”是否在心理承受范围内。若无法承受，请降低 $k$ 值。
4.  **保存配置**：点击“保存当前配置到组合”，以便在 Step 2 统一管理。

### Step 2: 多标的组合管理 (Portfolio)
**核心目的**：在多只股票之间分配有限的资金。

1.  **设定上限**：根据市场环境设置 **总仓位上限 (C_max)**（例如 1.0 表示满仓，1.5 表示上杠杆）。
2.  **归一化**：系统会根据各标的的 ERP 和波动率，自动计算权重，并按比例缩放，确保总风险敞口不超过设定的上限。

## 3. 关键参数设置指南

侧边栏的参数直接决定了策略的激进程度，请根据个人风格慎重设置。

### A. 估值锚点
* **$V_{target}$ (目标价)**：您认为合理的公允价值（如历史中位数 PE 对应的价格）。
* **$V_{hard}$ (硬底)**：极端悲观下的价格下限。**建议购买行权价 (Strike) 接近此价格的期权**，以物理锁定尾部风险。

### B. 策略约束
* **$k$ (Kelly Factor)**：资金安全倍数。
    * **0.5 (半凯利)**：**推荐默认值**。在长期增长和波动控制之间取得最佳平衡。
    * **1.0 (全凯利)**：仅建议在股价击穿 $V_{fill}$ 的极度低估时刻使用。
* **$\beta$ (Valuation Discount Factor)**：估值折扣系数。
    * **定义**：决定了当股价上涨接近目标价 ($V_{target}$) 时，仓位减少的速率。
    * **0.0 (死多头)**：即使股价到达目标价，Alpha 依然为 1.0，不做任何减仓。
    * **1.0 (极度保守)**：股价一旦到达目标价，Alpha 降为 0，建议完全清仓。
    * **0.2 ~ 0.5 (推荐)**：保留大部分底仓以享受趋势，仅做适度减仓。

---

## 4. 风险提示

1.  **GIGO 原则**：垃圾进，垃圾出。如果输入的 $V_{target}$ 过高或 Lambda 虚高，模型会建议极其危险的重仓。
2.  **黑天鹅风险**：凯利公式假设未来波动率与历史相似。若发生结构性崩盘（如财报暴雷），实际亏损可能远超模型预估。
3.  **杠杆双刃剑**：LEAPS 虽然没有追加保证金风险（Margin Call），但其价格波动剧烈。请务必关注 Step 1 中的“账户日波动率”指标。
